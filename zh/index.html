每一个 Claude Code & Codex
会话的归处。

不再翻 tmux 标签页、不再开八个 iTerm 窗口。 Quiver 把每一个终端 coding agent 会话 —— Claude Code、Codex，以及更多即将支持的 —— 都固定下来，本地或远程 SSH 主机都行：真实终端、真实 diff、逐块暂存、真实 shell，全在同一个画面里。

痛点

Coding agent 的会话总在四处流浪。

一会儿在 checkout-api 里和 Claude 改代码， 切到 storefront-web 让 Codex 验证一遍， 再开个 tmux 跑测试 —— 到周五下午，你已经忘了哪个会话在哪个分支上跑到哪了， 连是 Claude 还是 Codex 起的都得猜。

把 CLI 包成一个按钮一堆的界面不是答案 —— CLI 本身就是产品。 Quiver 不重写 Claude 或 Codex，它管理的是它们周围的那些窗口： 活的、置顶的、可搜索的，外加平时本来就要并排打开的那些东西。

能做什么

所有会话、所有项目，一个画面。

01 · 侧栏

跨项目，一眼看完。

所有 repo 下的会话按项目归类，状态点实时显示。常用项目置顶。 会话标题、首条 prompt、路径全部可搜索 —— 不管来自 Claude 还是 Codex。

02 · Agents

两个 agent，一个侧栏。

新建会话按项目选 agent。Claude C 和 Codex X 的会话共享同一份侧栏，按最近活跃排序，一键恢复。更多 agent 在路上。

03 · 终端

真实的 CLI，不是包壳。

真 PTY 跑的是和你 shell 里完全一样的 claude 或 codex —— Nerd Font、oh-my-zsh 主题、vim 快捷键、⌥ 跳词，全部正常。 下半屏 ⌃` 开一个 zsh pane，一个不够还能左右 分屏成多个。

04 · Diff & 文件

看清改动，再逐块暂存。

右侧自动跟随当前会话的工作目录。切到 Changes 看实时 git status，点任意文件弹出对比视图 —— 再像 VS Code 一样逐块暂存、取消暂存、丢弃。 文件树会把 git 忽略的条目灰显，还有 worktree 切换器跟随 agent 新建的分支。

05 · 远程

在任意 SSH 主机上跑 agent。

从你的 ~/.ssh/config（含 devpod）里挑一台主机， 浏览它的目录，直接在上面跑 Claude 或 Codex，完整终端就在 Quiver 里。 文件、Changes、diff、底部 shell 全部走同一条复用连接。 主机上有 tmux 时，断线也不丢会话 —— 重连回到同一个画面。

06 · 主题

浅色、深色，或你自己的。

内置浅色 / 深色主题任选（含 Catppuccin），也能直接导入一份 VS Code 配色主题文件 —— workbench 配色映射到应用界面， terminal.ansi* 映射到终端。导入的主题实时预览、重启后保留。

进入会话

任何对话一键恢复。

侧栏顶部的 OPENED 跟踪正在跑的会话， RECENT 让你不用记得"上周那个 bug 是在哪个目录里、用哪个 agent 讨论的"。 直接点，从上次结束的地方继续。

  • 会话直接读每个 agent 自己的日志文件 —— 没有私有协议，也没有后台 daemon。
  • 标题自动从对话里提取；每行带一个 agent 小标记，一眼区分 Claude 和 Codex。
  • 状态点反映真实状态：运行中、等用户输入、等待权限确认、空闲、已完成。

不用切来切去

你本来就要开的那个 shell，直接内置。

Quiver 不取代你的终端 —— 它把那个本来就跟 agent 并排开着的 shell 收编进来。 同一个 shell、同一个 cwd、同一份 dotfiles。 git diff 看 Claude 或 Codex 刚改的东西，跑测试、回答下一个 prompt， 全程不用 alt-tab。要两个？左右分屏即可。远程项目则直接在主机上、同一目录里开 shell。

本地或远程

在别人的机器上跑 agent。

一个 + 菜单，本地起会话，或在 ~/.ssh/config 里的任意主机上起（含 devpod）。 agent 跑在那边，完整终端、文件树、git diff 全部通过一条复用的 SSH 连接回传过来。

  • 远程项目在侧栏带一个 server 徽章，永远不会和本地项目搞混。
  • 主机上有 tmux 时，断线也不丢会话和正在跑的活， 重连直接回到同一个画面。

提交前先 review

一块一块地暂存。

先看清 agent 刚写了什么，再按自己的节奏提交。每一段连续改动都有独立的内联 Stage / Discard —— 留下好的那块、丢掉顺手多改的那行， 全程不离开窗口。文件列表像 VS Code 一样分成 Staged Changes 和 Changes，每个操作都直接落到 git apply —— 你暂存的，就是 git 看到的。

改成你的样子

浅色、深色，或你早就用惯的那套配色。

内置浅色 / 深色主题一键覆盖界面和终端。或者导入一份 VS Code 配色主题文件，Quiver 把它的 workbench 配色映射到应用、 terminal.ansi* 映射到终端 —— 你每天盯着的那套配色，处处一致。
